## 1. はじめに

### このシステムでできること

- プロジェクトごとに WBS（作業分解構造）を **最大3階層** で管理
- 各作業項目の **担当・期間・ステータス・工数・資料・リンク** を記録
- **ガントチャート** でスケジュールを一覧表示
- **担当者マスタ** の登録・CSV 一括入出力
- Supabase 連携時は **複数人でデータ共有**

### 2種類の「タスク」について

| 種類 | 場所 | 用途 |
|------|------|------|
| **WBS 作業項目** | WBS + ガントチャート | 階層構造の作業分解。詳細パネルでメタデータを管理 |
| **プロジェクトタスク** | タスク管理 | WBS とは独立したフラットな TODO リスト |

## 3. データの保存先

### localStorage モード（デフォルト）

- 設定不要ですぐ使える
- データは **そのブラウザにのみ** 保存される
- ブラウザのデータ削除で消える可能性あり

### Supabase モード（共有 DB）

セットアップ手順は [SUPABASE_SETUP.md](./SUPABASE_SETUP.md) を参照。

**移行:** Supabase 設定後、プロジェクト一覧に **localStorage から移行** ボタンが表示されます。以前ブラウザに保存したプロジェクトを DB へコピーできます。

### 保存タイミング

| 操作 | 保存 |
|------|------|
| WBS の編集 | 自動保存 |
| 作業詳細パネルの編集 | 400ms 後に自動保存 |
| プロジェクト作成・削除 | 即時保存 |
| リンクの追加 | **保存** ボタンまたは Enter で確定 |

## 4. マスター管理画面

### 4.1 プロジェクト一覧（`/`）

#### プロジェクトを作成する

1. **Add New** をクリック
2. プロジェクト名を入力（例: `新商品開発プロジェクト`）
3. 次のいずれかで作成:
   - **作成して WBS を編集** ボタンをクリック
   - **Enter を2回** 押す（1回目は確認、2回目で作成）

> Enter 1回だけでは保存されません。誤作成防止のため2段階になっています。

4. 作成後、WBS 編集画面へ自動遷移

#### プロジェクトを開く

- カードまたは表のプロジェクト名をクリック
- または **Open WBS** リンク

#### プロジェクトを削除する

1. 削除アイコン（🗑）をクリック
2. 確認ダイアログで **OK**

#### 検索・表示切替

- **Search Projects...** でプロジェクト名を検索
- **カード表示 / 表表示** を切り替え（設定はブラウザに記憶）

## 6. WBS + ガントチャート

### 6.1 画面レイアウト

```
┌─────────────────────┬──────────────────────┐
│  WBS 表（左 60%）    │  ガントチャート（右 40%）│
│  作業項目の編集       │  スケジュールの可視化    │
└─────────────────────┴──────────────────────┘
```

### 6.2 階層構造

| 階層 | コード例 | 説明 |
|------|---------|------|
| 親 | `1`, `2`, `3` | 大項目 |
| 子 | `1.1`, `1.2` | 中項目 |
| 孫 | `1.1.1`, `1.1.2` | 小項目（最大ここまで） |

コードは自動採番されます。手動での変更は不要です。

### 6.3 WBS 表の列

| 列名 | 内容 |
|------|------|
| ⠿ | ドラッグハンドル（並び替え用） |
| 作業項目 | コード + 名称 |
| 詳細 | 作業の説明（抜粋） |
| 担当 | 担当者名（自由入力） |
| 開始 | 開始日 |
| 終了 | 終了日 |
| 状態 | ステータス |
| 工数 | 作業工数 |
| 備考 | 補足メモ |
| 操作 | 各種ボタン |

### 6.4 作業項目の操作

| 操作 | 方法 |
|------|------|
| 名称を編集 | 作業項目名をクリック → 入力 → Enter またはクリックで確定 |
| 子項目を追加 | **＋**（子追加）ボタン |
| 同レベルに追加 | **＋**（同レベル追加）ボタン |
| トップレベル追加 | ツールバー **作業項目を追加** |
| 詳細を開く | **↗**（作業ページを開く）ボタン |
| 並び替え | 行左の ⠿ をドラッグ |
| 階層変更 | 行の中央付近にドロップすると子項目になる |
| 削除 | 削除ボタン（配下がある場合は確認あり） |

### 6.5 ツールバー

| ボタン | 機能 |
|--------|------|
| すべて展開 | 折りたたまれた項目をすべて開く |
| カテゴリを折りたたむ | 第1階層の子項目を折りたたむ |
| 作業項目を追加 | トップレベルに新規項目を追加 |

### 6.6 ガントチャート

- WBS 表の右側に表示
- 各作業項目の **開始日〜終了日** をバーで表示
- **今日の列** はアンバー色でハイライト
- **土日** は背景色が異なる
- 子項目に日付がない場合、子孫の日付範囲を集約表示

#### ステータスとバーの色

| ステータス | 色 |
|-----------|-----|
| 未着手 | グレー |
| 進行中 | 緑 |
| 完了 | 青 |
| 保留 | 黄 |

## 7. 作業詳細パネル

WBS 表の **↗** ボタン、または行クリックで右側に開きます。

### 7.1 パネルの開き方・閉じ方

| 操作 | 方法 |
|------|------|
| 開く | 行の **↗** ボタン |
| 閉じる | **✕** ボタン、オーバーレイクリック、**Escape** |
| 全画面 | **全画面表示** ボタン |
| 全画面を戻す | **Escape** または **サイドパネルに戻す** |

### 7.2 入力項目

#### 基本情報

| 項目 | 説明 |
|------|------|
| 詳細情報 | 作業の内容・目的 |
| ステータス | 未着手 / 進行中 / 完了 / 保留 |
| 担当者 | 担当者名（自由入力） |
| 開始日 | 作業開始日 |
| 終了日 | 作業終了日 |
| 作業工数 | 数値（例: 3 人日）。0.5 刻み |
| 対応期間 | 開始・終了日から自動表示 |

#### 資料・作業記録

- Markdown 風のテキストエリア
- 作成した資料の内容や作業ログを記載
- **全画面表示** で広い編集スペースを使える

#### 関連リンク

Google ドキュメントなど外部資料の URL を登録。

1. **リンクを追加** をクリック
2. **資料名** と **URL** を入力
3. **保存** をクリック（または URL 欄で Enter）
4. 保存後はコンパクト表示。**開く** で新タブで表示

> リンクは **保存** ボタンで確定します。パネルを閉じる前に保存してください。

#### 備考

短い補足や引き継ぎ事項。

#### 配下の作業項目

子ノード一覧。クリックでその項目のパネルに移動。

### 7.3 保存

- 入力内容は **400ms 後に自動保存**
- 右上に **保存中...** / **保存済み** と表示
- パネルを閉じる際も未保存分をフラッシュ保存

## 8. タスク管理

WBS とは **別データ** の TODO リストです。

### 8.1 タスクを追加する

1. サイドバー **タスク管理** を開く
2. **新規タスク** をクリック
3. 以下を入力:

| 項目 | 必須 | 説明 |
|------|------|------|
| 大項目 | | カテゴリ（過去入力から候補表示） |
| 項目 | ✅ | タスク名 |
| 詳細 | | 補足説明 |
| 優先度 | | 高 / 中 / 低 |
| 対応開始日 | | |
| 対応終了日 | | 開始日以降 |

4. **追加** をクリック

### 8.2 タスクを編集・削除

- 一覧の行をクリックして編集
- 削除ボタンで削除

## 9. アカウントマスタ

担当者情報を管理します（WBS の「担当」欄とは **未連携**。将来連携予定）。

### 9.1 アカウントを登録する

**方法1: 画面から**

1. `/account` → **Add New**
2. 表示名・メール・部署・役職を入力
3. **登録**

**方法2: CSV 一括登録**

1. `/account` → **一括登録**
2. CSV ファイルを選択

**CSV 形式:**

```csv
表示名,メール,部署,役職,権限
山田 太郎,taro@example.com,プロダクト開発部,PM,
```

- 表示名 **または** メールが必須
- 英語ヘッダー（`name`, `email`, `department`, `role`）も可

### 9.2 CSV 出力

**出力** ボタンで、現在の検索結果を `accounts-YYYY-MM-DD.csv` としてダウンロード。

## 12. よくある質問

### Q. データが他の PC に表示されない

**A.** 画面上部バナーを確認してください。

- **localStorage** 表示 → そのブラウザのみの保存。Supabase を設定してください
- **Supabase** 表示 → 同じ DB に接続しているか、環境変数が正しいか確認

### Q. Enter を押したら勝手にプロジェクトが作られた

**A.** 2026年7月以降、プロジェクト作成は **Enter 2回** で確定するよう変更されています。1回目は「もう一度 Enter で作成します」と表示されます。

### Q. WBS の担当者とアカウントマスタが連動しない

**A.** 現バージョンでは WBS の担当者は **自由入力テキスト** です。プロジェクト担当者管理画面も準備中です。

### Q. リンクを追加したのに消えた

**A.** リンクは **保存** ボタン（または URL 欄で Enter）で確定する必要があります。追加後に必ず保存してください。

### Q. ガントにバーが表示されない

**A.** 作業詳細パネルで **開始日** と **終了日** を設定してください。

### Q. Supabase で「テーブルが見つからない」エラー

**A.** Supabase SQL Editor で `supabase/schema.sql` を実行してください。詳細は [SUPABASE_SETUP.md](./SUPABASE_SETUP.md)。
