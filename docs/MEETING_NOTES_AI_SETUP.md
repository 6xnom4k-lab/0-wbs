# 議事録 AI 連携セットアップ

Notion などからコピーした議事録テキストを WBS 画面に貼り付け、OpenAI がタスク・状態・日付を提案し、承認後に WBS に反映する機能のセットアップ手順です。

## 1. OpenAI API キーを取得

1. [OpenAI Platform](https://platform.openai.com/) にログイン
2. **API keys** から新しいキーを作成

## 2. 環境変数を設定

`.env.local`（ローカル）および Vercel の Environment Variables に追加:

```env
OPENAI_API_KEY=sk-...
```

**注意:** `NEXT_PUBLIC_` プレフィックスは付けないでください（サーバー専用）。

## 3. 開発サーバーを再起動

```bash
npm run dev
```

## 4. 使い方

1. Notion 議事録ページを開き、本文をコピー
2. プロジェクトの **WBS + ガントチャート** 画面を開く
3. **議事録から AI 提案** をクリック
4. テキ録を貼り付け → **AI で解析**
5. 提案を確認・選択 → **選択を承認して WBS に反映**

## 提案の種類

| 種類 | 説明 |
|------|------|
| 新規 | 既存 WBS にないタスクとして追加 |
| 更新 | 既存 WBS 項目の状態・日付・担当などを更新 |

確度が **low** の提案はデフォルトで未選択です。内容を確認してから承認してください。

## トラブルシュート

| 症状 | 対処 |
|------|------|
| 「OPENAI_API_KEY が設定されていません」 | `.env.local` を確認し `npm run dev` を再起動 |
| 提案が0件 | 議事録に具体的なアクション項目・期限・担当が含まれているか確認 |
| 一部スキップ | WBS 3階層上限、日付形式、存在しない親 ID などが原因 |
| Vercel 本番で動かない | Vercel の Environment Variables に `OPENAI_API_KEY` を追加し再デプロイ |
